# Contributing

Thank you for your interest in contributing to this project! Here's how you can help:

## Getting Started

1. **Fork the repository** - Click the "Fork" button on GitHub
2. **Clone your fork** - `git clone https://github.com/your-username/rust-data-engineering-sandbox.git`
3. **Create a branch** - `git checkout -b your-feature-name`
4. **Make your changes** - Edit files as needed

## Making Changes

- Follow Rust conventions and best practices
- Write clear, descriptive commit messages
- Test your changes before submitting
- Keep changes focused and related to a single feature or fix

## Submitting Changes

1. **Commit your changes** - `git commit -m "Your descriptive message"`
2. **Push to your fork** - `git push origin your-feature-name`
3. **Open a Pull Request** - Go to GitHub and create a PR with a clear description
4. **Respond to feedback** - Update your PR based on review comments

## Code Standards

- Follow Rust style guidelines
- Use meaningful variable and function names
- Add comments for complex logic
- Ensure all tests pass

## Questions?

Feel free to open an issue if you have questions or suggestions!

Happy contributing! 🚀
